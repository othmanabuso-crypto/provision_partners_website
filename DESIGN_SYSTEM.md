# Provision Partners Design System

Version: 1.0

This document defines the complete visual language for the Provision Partners website.

Every component, page, and future feature must follow these standards.

---

# Design Philosophy

The website should communicate:

* Trust
* Professionalism
* Reliability
* Corporate Confidence
* Operational Excellence

Every design decision should reinforce these values.

Avoid designs that feel:

* Playful
* Trendy for the sake of trends
* Startup-focused
* Gaming-inspired
* E-commerce oriented

---

Corporate Blue

* Primary buttons
* Links
* Icons
* Headings
* Highlights

---

## Primary Hover

Used only for hover and active states.

---

## Accent

Corporate Red

Purpose:

* Small accents
* Notifications
* Important highlights
* Decorative elements

Primary Background

#FFFFFF

Dark Footer

---

# Typography

Font Family:

Inter

Hierarchy

H1

* Bold
* Large
* Strong visual impact

H2

* Bold
* Clear section heading

H3

* Semi-bold

Body

* Regular
* Comfortable reading

Captions

* Smaller
* Secondary color

Maintain generous line spacing throughout.

---

# Layout

Maximum Content Width

1280px

Section Padding

Desktop:

96px top & bottom

Tablet:

72px

Mobile:

56px

Content should always have generous whitespace.

---

# Grid

Desktop

4-column and 12-column layouts where appropriate.

Tablet

2-column layout.

Mobile

Single column.

---

# Cards

All cards across the website must use the same language.

Requirements

* Sharp edges
* Thin border
* Subtle shadow
* White background
* Smooth hover transition

Hover

* Slight upward movement
* Slightly stronger shadow

Rounded corners are not permitted unless explicitly approved.

---

Corporate Blue

White

Primary Hover

---

Secondary Button

---

Button Rules

All buttons must have:

* Equal height
* Equal padding
* Consistent typography
* Consistent spacing
* Same hover timing

---

# Icons

Style

Outline icons

Consistent stroke width

Use one icon family throughout the project.

Icons should visually align with accompanying text.

---

# Images

Preferred imagery

* Office supplies
* IT equipment
* Construction materials
* Warehouses
* Logistics
* Procurement
* Professional workplaces

Avoid

* Clipart
* Cartoons
* Abstract illustrations
* Fantasy AI art

Images should feel authentic and business-oriented.

---

# Animations

Animations should enhance usability, not distract.

Allowed

* Fade In
* Slide Up
* Slide Left
* Slide Right
* Subtle Scale
* Smooth Scroll

Avoid

* Bounce
* Shake
* Flash
* Rotation
* Excessive motion

Animation Duration

200–600ms

Animations should feel smooth and professional.

---

# Hero Section

The hero should be the strongest visual section.

Requirements

* Clean layout
* Strong typography
* Smooth entrance animations
* Professional imagery
* Clear call-to-action

The visitor should immediately understand:

"What does this company do?"

within the first five seconds.

---

# Trusted By Section

Use a continuously scrolling horizontal logo carousel.

Requirements

* Smooth infinite loop
* Equal logo spacing
* Consistent logo sizing
* Pause on hover (future enhancement)

---

# Navigation

Sticky navigation

Transparent at the top if appropriate.

Solid background after scrolling.

Navigation transitions should be subtle.

All navigation items should smooth-scroll to sections.

---

# Footer

Dark background

Four-column layout

Include

* Company
* Services
* Resources
* Contact

Social icons should match the project's icon style.

---

# Forms

Input fields

* Sharp edges
* Clear labels
* Visible focus state
* Professional spacing

Validation messages should be concise and helpful.

---

# Accessibility

Maintain sufficient color contrast.

Interactive elements must include visible focus states.

Use semantic HTML.

Keyboard navigation should be supported.

---

# Responsive Design

Desktop first.

Adapt gracefully to:

* Large desktop
* Laptop
* Tablet
* Mobile

No horizontal scrolling.

---

# Performance

Prefer optimized images.

Lazy-load non-critical content.

Avoid unnecessary JavaScript.

Reuse components whenever possible.

---

# Consistency Rule

If a new component is created, it must visually match the rest of the website.

Do not introduce new colors, spacing systems, button styles, shadows, or animation patterns without updating this document first.

This file is the visual source of truth for the Provision Partners website.
